# Incident Handler's Journal: Cybersecurity Response & Analysis

## Project Overview
This repository contains a comprehensive Incident Handler's Journal documenting the identification, analysis, and response to various cybersecurity incidents. Each entry follows a structured approach to capture the "5 W's" (Who, What, Where, When, Why) and outlines technical observations, root cause analyses, and remediation recommendations.

### Incident 3: Web Application Vulnerability & Cyber-Extortion
* [cite_start]**Description:** A complex attack involving forced browsing and data exfiltration at a financial organization[cite: 4].
* **Root Cause:** A vulnerability in an e-commerce web application allowed for a forced browsing attack. [cite_start]Attackers accessed customer transaction data by modifying order numbers in the URL string[cite: 4].
* [cite_start]**Impact:** Thousands of purchase confirmation pages were compromised, resulting in approximately $100,000 in direct costs to the organization[cite: 4, 5].
* [cite_start]**Response Activity:** On-site investigation, detailed analysis of web application access logs, and confirmation of data exfiltration[cite: 4, 5].
